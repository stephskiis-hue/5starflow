# CLAUDE.md — 5StarFlow

Static HTML/CSS/JS site. No framework, no build tool. Read the context files before coding.

## Core Rules

1. **No over-engineering.** No npm, no bundler, no framework — ever.
2. **Edit, don't create.** Prefer editing existing files over making new ones.
3. **Read before editing.** Never assume the current state of a file.
4. **Mobile-first.** All UI changes must hold up below 375px.
5. **No hardcoded colors.** CSS variables only.
6. **No trailing summaries.** Don't summarize what you just did.

## Quick Reference

- **Brand:** dark green (`#1A3C2E`) + gold (`#D4A017`) + off-white (`#F9F7F2`)
- **Font:** Inter, Google Fonts
- **Container:** max-width 1100px, padding 0 1.5rem
- **CSS files:** `css/styles.css` (global) · `css/pricing.css` (pricing page + nav dropdown)
- **JS:** `js/main.js` — nav, FAQ accordion, scroll reveal, ROI calculator
- **Contact:** hello@5starflow.com (all CTAs — no payment processor yet)
- **Nav dropdown CSS** is in `pricing.css`, not `styles.css`

## After Any HTML or CSS Edit

Run the `ui-verifier` agent — it takes desktop + mobile screenshots using headless Chrome, inspects them, and re-edits if anything looks broken. Screenshots saved to `.claude/screenshots/`.

## Before Starting Any Task

1. Check `context/css-reference.md` for existing classes and section order
2. Check `context/changelog.md` to know what's already done
3. Check `context/placeholders.md` before touching any link or form
